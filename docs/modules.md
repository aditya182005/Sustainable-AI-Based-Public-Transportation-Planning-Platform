# EcoTransit AI – Development Modules

1. Setup
2. Backend architecture
3. Database schemas
4. Seed data
5. Auth
6. Frontend layout
7. Maps integration (basic)
8. ML models (baseline)
9. Prediction APIs
10. Route optimization (basic)
11. Dashboard pages (core views)
12. Sustainability analytics
13. Reports (export/scheduled)
14. Chatbot (Q&A)
15. i18n
16. Deployment
17. Testing (cross-cutting)

## Module Descriptions
- **Setup**: Environment, dependencies, folder structure, docs.
- **Backend architecture**: Express server, routes, middleware.
- **Database schemas**: MongoDB collections for users, routes, predictions, reports.
- **Seed data**: Sample users, Hyderabad routes/stops, demo predictions.
- **Auth**: JWT, password hashing, role-based access.
- **Frontend layout**: React + Tailwind base, routing, auth context.
- **Maps integration**: Leaflet + OpenStreetMap, basic route display.
- **ML models**: Passenger demand, delay, occupancy, sustainability formulas.
- **Prediction APIs**: Flask endpoints, Node.js proxy routes.
- **Route optimization**: Graph algorithms + ML scoring.
- **Dashboard pages**: KPI cards, charts, insights, 15-page skeleton.
- **Sustainability analytics**: CO₂, fuel usage, efficiency metrics.
- **Reports**: PDF export with charts, KPIs, insights.
- **Chatbot**: Rule-based Q&A, multilingual support.
- **i18n**: React-i18next setup, language switcher.
- **Deployment**: Vercel, Render/Railway, MongoDB Atlas.
- **Testing**: Unit, integration, frontend component tests.
